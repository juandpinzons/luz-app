# ENGINEERING PRINCIPLES

1. Architecture before implementation.
2. Verify before acting.
3. Domain before framework.
4. Documentation is part of the product.
5. One owner per decision.
6. Small, reversible changes.
7. Security by default.
8. Optimize after correctness.
9. LLMs assist; they do not own system state.
10. Build for the next decade, not the next demo.
