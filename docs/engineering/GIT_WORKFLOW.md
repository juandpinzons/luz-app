# GIT WORKFLOW

## Branches

main
- Production only.

develop
- Integration branch.

feature/<name>
- One feature per branch.

hotfix/<name>
- Critical fixes.

## Commits

Conventional Commits:

feat:
fix:
refactor:
docs:
test:
chore:

## Pull Request Flow

1. Claude opens PR.
2. Codex reviews.
3. CTO approves architecture.
4. Merge into develop.
5. Release to main by version tag.
